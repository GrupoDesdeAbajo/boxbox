javascript physics made easy

what is boxbox?

boxbox is a JavaScript library that makes it easier to use the box2d physics engine.

no, i mean really, what is it

boxbox is an API wrapper for a JavaScript port of an ActionScript port of a C++ program.

The box2d physics engine is hard to use for a beginner. Further, its C++ style API doesn't make sense to JavaScript programmers. boxbox tries to solve both problems.

features

  • simple API
  • work with entities rather than fixtures, bodies, and shapes
  • attach events to entities rather than the whole world
  • reuse common configuration among similar entities
  • built in render layer with image support

api reference

In progress. I'll make this a lot better when the API stabilizes.

boxbox

Global boxbox object.

boxbox.createWorld

Create a new world. A world is a single stand-alone physics simulation. It may contain any number of entities. It starts out empty.

Parameters:

  1. canvas (required) - canvas element to render on
  2. options
    • debugDraw - render debugging shapes on canvas. default false
    • gravity - force of downward gravity. can be negative. default 10
    • scale - if rendering is used, rendering scale in pixels per meter. default 30

var canvas = document.getElementById('myCanvas'); var world = boxbox.createWorld(canvas); var canvas = document.getElementById('myCanvas'); var world = boxbox.createWorld(canvas, { gravity: 20, scale: 60 });

world

A stand-alone physics simulation. Contains entities and renders out on a single canvas.

world.createEntity

Create a new entity. An entity is a single object in the world. It interacts with other entities and is rendered to the canvas.

Parameters:

  1. options (all are technically optional and have defaults)
    • name - entity name, useful in event handlers
    • x - starting x coordinate
    • y - starting y coordinate
    • shape - square, circle, or polygon. default square
    • type - dynamic or static. static entities can't move. default dynamic
    • width - for squares, in meters
    • height - for squares, in meters
    • radius - for circles, in meters
    • points - for polygons*
    • density - weight is calculated based on this and size. default 2
    • friction - affects object interactions. default 1
    • restitution - bounciness. default .2
    • active - participates in collisions and dynamics. default true
    • fixedRotation - object can't rotate. default false
    • bullet - perform expensive continuous collision detection. default false
    • image - path to image for rendering. default none
    • imageOffsetX - adjust image rendering location
    • imageOffsetY - adjust image rendering location
    • imageStretchToFit - stretch image based on entity size
    • draw - custom draw function for this entity

var player = world.createEntity({ name: 'Player', x: 5, y: 10, shape: 'square', width: 3 height: 6 });

*Points look like this. They must go clockwise and they most produce a convex polygon.

world.createEntity({ shape: 'polygon', points: [{x: 0, y: 0}, {x: 1, y: 1}, {x: 0, y: 1}] });

entity

A single entity in a world. Is affected by forces, gravity, joints, and other entities.

entity.destroy

Destroy this entity and remove it from the world.

entity.destroy();

entity.applyImpulse

Apply an instantaneous force to an entity.

Parameters:

  1. power - force of the impulse
  2. direction - direction of the force. if one argument it's degrees. if two arguments it's a vector

// defaults to 0 degrees, which is up entity.applyImpulse(10); // 180 degrees is down entity.applyImpulse(10, 180); // use a vector instead of degrees. 1,-1 is 45 degrees entity.applyImpulse(10, 1, -1);
